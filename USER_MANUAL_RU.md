# Script Runner - Руководство пользователя

## Обзор

Script Runner — это веб-приложение для управления и выполнения shell-скриптов. Оно предоставляет современный интерфейс для создания, редактирования, организации и запуска shell-скриптов с мониторингом вывода в реальном времени и отслеживанием истории выполнения.

## Основные функции

### 1. Управление скриптами

- **Навигация по файловому дереву**: Просмотр и организация shell-скриптов в иерархической структуре папок
- **Создание скриптов**: Добавление новых shell-скриптов (файлы `.sh`) и папок
- **Редактирование скриптов**: Встроенное редактирование с подсветкой синтаксиса
- **Переименование и перемещение**: Переименование файлов/папок и перемещение их между директориями
- **Удаление**: Удаление скриптов и папок (с диалогами подтверждения)
- **Автосохранение**: Изменения автоматически сохраняются на сервере

### 2. Выполнение скриптов

- **Запуск скриптов**: Выполнение shell-скриптов одним кликом
- **Вывод в реальном времени**: Просмотр вывода скриптов и ошибок по мере их появления
- **Статус выполнения**: Отображение состояний выполнения (запущен, завершен, неудачен)
- **Прерывание выполнения**: Остановка запущенных скриптов при необходимости
- **Результаты выполнения**: Четкие индикаторы успеха/неудачи

### 3. Навигация и представления

Приложение имеет три основных представления, доступных через боковую панель:

#### Представление "Скрипты"

- **Основное рабочее пространство** для управления и редактирования скриптов
- **Файловое дерево** слева для навигации
- **Редактор/просмотрщик скриптов** справа при выборе скрипта
- **Кнопки действий** для редактирования и запуска скриптов

#### Представление "Активные"

- **Мониторинг в реальном времени** запущенных в данный момент скриптов
- **Обновления статуса в реальном времени** с прогрессом выполнения
- **Счетчик-бейдж** показывающий количество активных выполнений
- **Быстрый доступ** к деталям запущенных скриптов

#### Представление "История"

- **История выполнения** всех завершенных запусков скриптов
- **Детали выполнения** включая время начала, продолжительность и результаты
- **Просмотр вывода** для прошлых выполнений
- **Счетчик непросмотренных** для новых результатов выполнения
- **Индикаторы статуса** успеха/неудачи

### 4. Функции пользовательского интерфейса

- **Современный дизайн**: Чистый, адаптивный интерфейс с темной темой
- **Контекстные меню**: Действия по правому клику для операций с файлами
- **Функция поиска**: Глобальный поиск по приложению
- **Всплывающие подсказки**: Полезная информация при наведении
- **Диалоги подтверждения**: Защитные запросы для деструктивных действий
- **Горячие клавиши**: Быстрый доступ к общим функциям

### 5. Мониторинг выполнения

- **Живой вывод**: Отображение вывода скриптов и ошибок в реальном времени
- **Время выполнения**: Отслеживание времени начала и продолжительности
- **Индикаторы статуса**: Визуальная обратная связь для состояний запущен, успех, неудача и прерван
- **История вывода**: Просмотр полного вывода из предыдущих выполнений
- **Обработка ошибок**: Четкое различие между обычным выводом и сообщениями об ошибках

## Начало работы

1. **Запуск приложения**: Запустите сервер разработки с помощью `bun dev`
2. **Навигация по скриптам**: Используйте файловое дерево в представлении "Скрипты" для просмотра существующих скриптов
3. **Создание скрипта**: Правый клик в файловом дереве для создания новых скриптов или папок
4. **Редактирование скриптов**: Нажмите кнопку "Редактировать" для изменения содержимого скрипта
5. **Запуск скриптов**: Нажмите кнопку "Запустить" для выполнения скриптов
6. **Мониторинг выполнения**: Переключитесь в представление "Активные" для просмотра запущенных скриптов
7. **Просмотр истории**: Проверьте представление "История" для результатов прошлых выполнений

## Ключевые рабочие процессы

### Создание и запуск скрипта

1. Перейдите в представление "Скрипты"
2. Правый клик в файловом дереве → Создать новый скрипт
3. Введите имя скрипта (расширение `.sh` добавляется автоматически)
4. Нажмите "Редактировать" для добавления содержимого скрипта
5. Нажмите "Сохранить" для сохранения изменений
6. Нажмите "Запустить" для выполнения скрипта
7. Следите за выводом в реальном времени

### Управление организацией скриптов

1. Создавайте папки правым кликом в файловом дереве
2. Перетаскивайте скрипты между папками
3. Переименовывайте файлы/папки правым кликом и выбором переименования
4. Удаляйте ненужные файлы с подтверждением

### Мониторинг выполнений

1. Переключитесь в представление "Активные" для просмотра запущенных скриптов
2. Нажмите на имена скриптов для перехода к редактору скриптов
3. Просматривайте вывод и статус выполнения в реальном времени
4. Проверяйте представление "История" для завершенных выполнений

## Технические заметки

- Скрипты хранятся и управляются на сервере
- Связь в реальном времени через Server-Sent Events (SSE)
- Поддержка стандартного выполнения shell-скриптов
- Автоматическая синхронизация файлов между клиентом и сервером
- Построено на React, TypeScript и современных веб-технологиях

## Советы

- Используйте описательные имена для ваших скриптов
- Организуйте скрипты в логические структуры папок
- Следите за представлением "Активные" для долго выполняющихся скриптов
- Регулярно проверяйте представление "История" для результатов выполнения
- Используйте функцию поиска для быстрого поиска конкретных скриптов
